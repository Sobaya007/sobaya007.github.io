    Arch Linuxインストールメモ

    Posted on 2019-12-08 · 344 words. · 2 minute read.
    環境構築
    環境構築 · OS

    もう完全に自分用メモ書き。
    ちなみに私はUEFIブートでWin10とArchのデュアルブートにしています。
    CPUはintel製。
    GPUはNvidia製。
    ブートマネージャはsystemd-bootです。
    デスクトップ環境はgnomeです。
    インストール作業中の通信はWi-Fiでした。
    これがベストかはわからないが、できたので良し。
    様々なところを参考にしましたが、どこも通しでは参考にしていません。

    1. Live USBを作る。 私はrufusでやりました。
    2. BIOSに行ってSecure Bootを切る。
    3. Windowsとデュアルブートする場合は高速スタートアップを切る
    4. Arch用のパーティションを切る
    5. Live USBを差してBIOSへ行き、Bootの優先順位のところでUSBっぽいのを最優先にする
    6. Live USBからArchを起動
    7. lsblkで各パーティションの名前を確認 bash nvme0n1 259:0 0 477G 0 disk ├─nvme0n1p1 259:1 0 260M 0 part /boot ├─nvme0n1p2 259:2 0 16M 0 part ├─nvme0n1p3 259:3 0 257.6G 0 part ├─nvme0n1p4 259:4 0 218.1G 0 part / └─nvme0n1p5 259:5 0 1000M 0 part こんなかんじのノリでした。ここの名前は結構デバイスによって違う。

    8. parted -lでパーティションの役割を確認

      1    1049kB  274MB  273MB   fat32             EFI system partition          boot, hidden, esp
2    274MB   290MB  16.8MB                    Microsoft reserved partition  msftres
3    290MB   277GB  277GB   ntfs              Basic data partition          msftdata
4    277GB   511GB  234GB   ext4              Basic data partition          msftdata
5    511GB   512GB  1049MB  ntfs              Basic data partition          hidden, diag

      こんなかんじになっていた。
      1番がEFIパーティション。
      2番がWindows用の謎領域。
      3番がWindows10。
      私はWindows10のメインパーティションを縮小してその後ろに空き領域を作ったので、4番の領域がArch用となった。
      ということで以下EFIパーティションをnvme0n1p1、Arch用パーティションをnvme0n1p4としてしまうので、適宜そのときの状況に合わせて読み替えてください。

    9. mkfs.ext4 /dev/nvme0n1p4でArch用パーティションをフォーマット

    10. mount /dev/nvme0n1p4 /mntでArch用パーティションをマウント

    11. mkdir /mnt/bootでArch用パーティション内にEFIパーティションのマウントポイントを作成

    12. mount /dev/nvme0n1p1 /mnt/bootでEFIパーティションをマウント

    13. wifi-menuでwifiにつなぐ

    14. timedatectl set-ntp trueでシステムクロックを合わせる これが要るのかは私にはわからない

    15. vim /etc/pacman.d/mirrorlistでpacmanの使うミラーを選択する。
      具体的にはJapanっぽいのを適当に選んで一番上の行に持っていけばよい。

    16. pacstrap /mnt base linux linux-firmwareでディレクトリ構築+ベースシステムのインストール

    17. genfstab -U /mnt >> /mnt/etc/fstabでfstabを生成 よくわかっていないけどこれがないとあとで起動したときにマウントの仕方がわからなくなるっぽい。

    18. chroot /mntで作ったシステムに侵入

    19. ln -sf /usr/share/zoneinfo/Asia/Tokyo /etc/localtimeでタイムゾーンを東京にする

    20. hwclock --systohc --utcでなんか必要っぽいファイルを生成

    21. vim /etc/locale.genでen_US.UTF-8 UTF-8とja_JP.UTF-8 UTF-8のコメントを外す

    22. locale-genでlocaleっぽいファイルを生成

    23. echo LANG=en_US.UTF-8 > /etc/locale.confで使用するlocaleを設定。
      ちなみに私はここでtypoしたためにめっちゃ面倒を被りました。

    24. echo KEYMAP=jp106 > /etc/vconsole.confでキー配置を日本語用に。

    25. 適当にコンピュータの名前を決めてecho hoge > /etc/hostname 私はecho sobaya > /etc/hostnameとしました。

    26. vim /etc/hostsで中身を以下のようにする。

      127.0.0.1   localhost
::1     localhost
127.0.1.1   sobaya.localdomain  sobaya

    27. pacman -S iputils netctl dhcpcd iw wpa_supplicant dialogでネット系のいろいろを入れる
      これで必要十分かは不明だが、とりあえずこんだけ入れとけばなんとかなる

    28. mkinitcpio -Pする

    29. passwdでrootパスワードを設定

    30. bootctl --path=/boot installでEFIパーティションにsystemd-bootを突っ込む

    31. vim /boot/loader/loader.confで以下のようなファイルを作成

      default  arch
timeout  4
editor   no

    32. blkid /dev/nvme0n1p1 > /boot/loader/entries/arch.confで適当に前準備

    33. vim /boot/loader/entries/arch.confで以下のようなファイルを作成

      title   Arch Linux
linux   /vmlinuz-linux
initrd  /intel-ucode.img
initrd  /initramfs-linux.img
options root=PARTUUID=hogehoge-foo-baz-hogehogehoge rw

      hogehogeのところにはすでに書き込まれていた部分を使えば楽。

    34. pacman -S intel-ucodeでintel-ucode.imgを突っ込む

    35. exitで一段抜ける

    36. umount -R /mntでアンマウント

    37. shutdown nowで電源を落とす

    38. Live USBを引っこ抜いて電源を入れる

    39. systemd-bootが立ち上がるので、Arch Linuxを選択(or放置)

    40. ログインを求められるので、ユーザー名rootでログイン

    41. wifi-menuでもっかいWi-Fiに接続

    42. pacman -S gnome gnome-extraでデスクトップ環境を突っ込む

    43. systemctl enable gdmでgdmを登録

    44. pacman -S nvidiaでnvidiaドライバを突っ込む

    45. rebootで再起動

    46. gnomeが立ち上がるので、ユーザー名rootでログイン

    47. Superキー(Windowsキー？)を押してtermとか打つとterminalが選択できるので、起動

    48. systemctl enable NetworkManagerでNetworkManagerを登録
      これをしないとgnomeのGUIでWi-Fiが設定できない

    49. 右上のほうからWi-Fiを接続

    50. useradd -d /home/sobaya -s /bin/bash -m sobayaとかでユーザー作成
      ちなみに私はfish派なのでpacman -S fishしてから-s /usr/bin/fishしました。 あとなぜかfishの場所を見ようと思ってwhich fishしたらwhichが入っていなかったのでpacman -S whichしました。なぜwhichがないんだ。。。

    51. passwd sobayaでパスワード設定

    52. usermod -aG wheel sobayaでwheelグループ(sudoできるグループに登録)

    53. pacman -S neovim sudo

    54. nvim /etc/sudoersでwheelの行のコメント解除

    55. 一度ログアウトして、一般ユーザーで再度ログイン
      私はここでnvidiaの魔の手にかかり、ログインループ現象をくらいました。
      解決策として、ログイン画面の歯車ボタンを押してXorgのgnomeからログインするという手があります。 どうもWaylandと相性が悪いらしい。。。

    56. sudo pacman -S base-devel git go

    57. git clone https://aur.archlinux.org/yay.git

    58. cd yay

    59. makepkg -siでyayをインストール

    60. yay -S google-chromeでchromeをインストール

    61. sudo pacman -S noto-fonts-cjkでとりあえず日本語フォントをインストール

    62. sudo pacman -S fcitx-mozc fcitx-gtk3 fcitx-qt5で日本語入力をインストール

    63. Super押してSettingで歯車を召喚し、「地域と言語」を「日本語」「日本」「日本語」にする

    64. Super押してfcitxでペンギン(設定のほう)を召喚し、入力メソッドがMozcになっているのを確認

    65. sudo pacman -S dockerでdockerをインストール

    66. systectl enable dockerでdockerを登録

    67. systemctl start dockerでdockerdを起動

    68. usermod -aG docker sobayaでユーザーをdockerグループ追加

    69. newgrp dockerでdockerグループを有効化

    70. docker infoが出たら優勝

    ここまで来ると

    • デスクトップ環境
    • ブラウザ
    • 日本語表示/入力
    • Nvidiaドライバインストール
    • 一般ユーザー(sudoerかつdocker権限持ち)の作成

    まで行っているのでどうにかなるでしょう。
